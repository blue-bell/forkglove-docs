---
title: Built-in types
description: Primitive types are the building blocks of Foxglove's supported message schemas.
---

原始类型是Foxglove支持的消息模式的构建块。

消息模式中的每个字段都有一个类型。这个类型可以是另一个消息模式、一个枚举或下面记录的原始类型之一：

### `boolean`

`true` 或 `false`。

### `bytes`

原始二进制数据，在JavaScript中表示为 `Uint8Array`。

### `enum`

作为一组命名常量中的键的数字。

### `float64`

64位浮点数。

### `string`

以UTF-8编码的字符串值。

### `time`

| 字段    | 类型   | 必需 | 描述                        |
| ------- | ------ | ---- | --------------------------- |
| `sec`   | uint32 | ✓    | 自纪元以来的秒数            |
| `nsec`  | uint32 | ✓    | 自纪元以来的额外纳秒数      |

_注意: Foxglove的[Protobuf模式](/docs/visualization/message-schemas/introduction#protobuf-and-json-schema)使用[`google.protobuf.Timestamp`](https://protobuf.dev/reference/protobuf/google.protobuf/#timestamp)表示`time`值，它使用`seconds`和`nanos`字段。然而，在[用户脚本](/docs/visualization/panels/user-scripts)、[消息转换器](/docs/visualization/extensions/introduction#message-converters)和Foxglove的其他部分中，这些值将显示为`sec`和`nsec`字段，以便与其他数据格式保持一致。_

### `duration`

| 字段    | 类型   | 必需 | 描述                                  |
| ------- | ------ | ---- | ------------------------------------- |
| `sec`   | int32  | ✓    | 秒偏移量                              |
| `nsec`  | uint32 | ✓    | 正方向的纳秒偏移量                    |

_注意: Foxglove的[Protobuf模式](/docs/visualization/message-schemas/introduction#protobuf-and-json-schema)使用[`google.protobuf.Duration`](https://protobuf.dev/reference/protobuf/google.protobuf/#duration)表示`duration`值，它使用`seconds`和`nanos`字段。然而，在[用户脚本](/docs/visualization/panels/user-scripts)、[消息转换器](/docs/visualization/extensions/introduction#message-converters)和Foxglove的其他部分中，这些值将显示为`sec`和`nsec`字段，以便与其他数据格式保持一致。_

### `uint32`

0到2<sup>32</sup>&minus;1之间的非负整数值。

### `int32`

&minus;2<sup>31</sup>到2<sup>31</sup>&minus;1之间的整数值。
